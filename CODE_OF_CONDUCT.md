# Grimoire Code of Conduct

Grimoire is a local-first tool for private thought, writing, vaults, and agent
work. Public project spaces must stay useful, kind, and safe enough for people
to report issues without exposing private material.

## Expected Behavior

- Be respectful and direct.
- Assume good faith, but do not ignore harm.
- Keep bug reports, pull requests, and discussions focused on reproducible
  project work.
- Respect privacy. Do not ask people to paste private vault content, secrets,
  credentials, personal journals, diary entries, dream notes, or private
  attachments into public issues.
- Use the right public channel: bugs in GitHub Issues, features in Canny,
  source/setup help through [SUPPORT.md](SUPPORT.md), and vulnerabilities
  through private security reporting.
- Accept maintainer decisions about scope, release readiness, licensing,
  trademark use, and private-data removal.

## Unacceptable Behavior

- Harassment, threats, insults, stalking, or sustained unwanted attention.
- Sexualized, discriminatory, or demeaning language or imagery.
- Publishing someone else's private information without explicit permission.
- Posting secrets, credentials, private vault content, private journals, diary
  entries, dream notes, or private attachments.
- Derailing issues or pull requests with unrelated arguments, spam, or repeated
  demands after a maintainer decision.
- Misrepresenting browser source mode as proof of native file IO, packaged
  installers, update feeds, or platform support.

## Scope

This code applies in Grimoire project spaces, including GitHub issues, pull
requests, discussions, documentation, release channels, and official project
support surfaces.

Maintainers may remove content, close threads, block accounts, or restrict
participation when behavior makes the project less safe or less useful.

## Reporting

For conduct concerns, email **removed** with the subject line
**`[Grimoire Conduct]`**.

For security vulnerabilities, follow [SECURITY.md](SECURITY.md) instead and use
the subject line **`[Grimoire Security]`**.

Reports should include:

- what happened
- where it happened
- links or screenshots if available
- whether private information or secrets were exposed

Do not include unnecessary private vault content or credentials in the report.

## Maintainer Response

Maintainers will review reports, remove private or harmful material when
appropriate, and choose a response proportional to the issue. Possible responses
include clarification, warning, content removal, issue or PR locking, temporary
restrictions, or permanent blocking.

The goal is not ceremony. The goal is a project space where people can help make
Grimoire better without turning public infrastructure into a liability.
